# storage/postgres/ — Design

## Patterns

- **sqlx** with named parameters for all queries — prevents SQL injection by design.
- **Repositories** are struct types that hold a `*sqlx.DB` reference.
- **Migrations** are embedded in the binary via `embed.FS` and run at startup.
- All repo methods take `context.Context` as first parameter.
- UUID primary keys generated by PostgreSQL (`gen_random_uuid()`).
- Soft-delete for listings (status='deleted'), hard-delete for other entities where appropriate.

## Scanning

Using `sqlx.Get` / `sqlx.Select` with `db` struct tags on domain types.
